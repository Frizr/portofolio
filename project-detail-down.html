Project Overview

A Godot 4 C# isometric action-combat game project with custom projection, responsive movement, combat mechanics, enemy behavior, and game state architecture.

Problem

Laundry businesses need a structured way to manage daily transactions, customer orders, payments, reports, and shop settings. Manual workflows can make it harder to track order status, payment state, and operational summaries.

Goal

The goal is to create a practical POS system with a clean interface and core operational features for order handling, payment tracking, reporting, and configuration.

My Role

I worked on the web application structure, user interface pages, front-end interaction flow, and backend integration direction for the laundry POS workflow.

Tech Stack

Godot 4 C# .NET Python

Main Features

  • Custom isometric vector math
  • 8-directional movement
  • Dodge roll and invincibility frames
  • Hitbox and combo combat system
  • Enemy behavior state machine
  • Game manager, HUD, and screen feedback

Development Process

The process focused on mapping the main laundry workflow first: login, dashboard, orders, payments, reports, and settings. After that, each page was structured into reusable interface patterns and connected with PHP and MySQL concepts.

Challenges

The main challenge was keeping the system understandable while still covering multiple operational needs such as authentication, order management, payments, reporting, and configuration.

Result

The result is a structured laundry POS web application repository with a clear page set, core business workflow, database-oriented backend direction, and a responsive UI foundation.

What I Learned

I learned how to organize a practical web application, connect front-end pages to backend concepts, design order and payment workflows, and document a project in a way that is easier to explain on a project detail page.
